Config

Overview

The config library allows easy access to a configuration array. Furthermore it has the following options

  1. Autoload configuration file and return an array
  2. Instance support and easy syntax. ie. Config::Router();
  3. Cascading, cascade the configuration options by inheritance.
  4. Supports default values
  5. Support Grouping, then return different results based upon which group is set. (Great for environment variables)
  6. Full Default, Group and Cascade integration.
  7. Easy to implement, just include use the trait.

Overview

The code is split up into the following classes

  1. Library.php : The actual config library, useful for extending functionality.
  2. Instance.php : Instance of the object class.
  3. Mixin\Library.php : The mixin to use with in your config classes..

Usage 


    // The configuration class
    class Mail extends \Hive\Config\Library
    {
        use \Hive\Config\Mixin\Library;

        public static $default = [
            'subject'   => 'Wax',
            'from'      => 'wax@hive.com.au',
        ];
    }

    print_r(\Hive\Config\Instance::Mail());

Outputs


Array
(
    [subject] => 'Wax'
    [from] => 'wax@hive.com.au'
)

Grouping with Default Values 


    // The configuration class
    class Mail extends \Hive\Config\Library
    {
        use \Hive\Config\Mixin\Library;

        public static $default = [
            'subject'   => 'Wax',
            'from'      => 'wax@hive.com.au',
        ];
        public static $development = [
            'from'      => 'developer@hive.com.au',
        ];
    }


    /**
     * As the group is being set on the instance, this will remain the 'group' for all
     * configuration instances, until changed.
     */
    $config         = new Config\Instance();
    $config::$group = 'development';
    print_r( $config::Mail() );

Outputs


Array
(
    [subject] => 'Wax'
    [from] => 'developer@hive.com.au'
)

Limitations with in php5, will cause a colliding constructor definitions with a trait named __constructor. There is an easy solution, you just have to alias the trait



    class Mail extends \Hive\Config\Library
    {
        // Use an alias for the constructor
        use \Hive\Config\Mixin\Library
        {
            \Hive\Config\Mixin\Library::__construct as private construct;
        }

        public static $default = [
            'subject'   => 'Wax',
            'from'      => 'wax@hive.com.au',
        ];

        // Call the alias constructor
        public function __construct($group, array $values = [])
        {
            self::construct($group, $values);
        }

    }

Advance

The config library has many many more options and features, to view them please see our Examples.

Notes about inheriting.

  1. It is possible to inherit a config class, in which case the settings from the parent will also be inherited.
  2. Items will give preference to themselves over parents, ie. object/$sever > object/$default > parent/$server > parent/$default
  3. At the moment, the child is required to declare the default property if the parent uses it, otherwise it will override.
